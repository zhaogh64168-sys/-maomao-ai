# 毛毛AI Streamlit 助手

一个基于 Streamlit、OpenAI 和 Supabase 的个人 AI 助手，支持密码登录、多用户隔离、聊天记录、长期记忆、图片分析、模型切换和使用统计。

## 功能

- 密码登录
- 多用户：不同密码对应不同 `user_id`
- OpenAI 聊天
- 图片上传分析
- GPT-5 / GPT-5-mini / GPT-5-nano 模型切换
- Supabase 保存、读取、清空 `chat_history`
- Supabase 保存、读取、删除、清空 `memories`
- 输入 `记住xxx` 自动保存长期记忆
- 输入 `忘记xxx` 自动删除相关长期记忆
- 联网搜索开关，已预留 Tavily / SerpAPI / Bing Search 接口位置
- OpenAI token 和费用统计基础日志
- 页面显示今日请求次数和今日总 tokens

## Streamlit Secrets

在 Streamlit Cloud 的 Secrets 中配置：

```toml
APP_PASSWORD = "你的访问密码"
OPENAI_API_KEY = "你的 OpenAI API Key"
SUPABASE_URL = "https://你的项目.supabase.co"
SUPABASE_KEY = "你的 Supabase anon key 或 service role key"
```

单用户时，`APP_PASSWORD` 直接写一个密码即可。应用会使用默认用户 `default_user`。

多用户时，`APP_PASSWORD` 可以写成 JSON 字符串，格式是 `user_id -> password`：

```toml
APP_PASSWORD = '{"maomao":"password1","alice":"password2","bob":"password3"}'
OPENAI_API_KEY = "你的 OpenAI API Key"
SUPABASE_URL = "https://你的项目.supabase.co"
SUPABASE_KEY = "你的 Supabase anon key 或 service role key"
```

不要把任何 API Key 写进代码，也不要把完整密钥显示在页面上。

## Supabase 数据库 SQL

在 Supabase SQL Editor 中执行以下 SQL：

```sql
create table if not exists public.chat_history (
  id bigint generated by default as identity primary key,
  user_id text not null,
  role text not null check (role in ('user', 'assistant')),
  content text not null,
  created_at timestamptz not null default now()
);

create index if not exists chat_history_user_id_id_idx
  on public.chat_history (user_id, id);

create table if not exists public.memories (
  id bigint generated by default as identity primary key,
  user_id text not null,
  memory text not null,
  created_at timestamptz not null default now()
);

create index if not exists memories_user_id_id_idx
  on public.memories (user_id, id);

create table if not exists public.usage_logs (
  id bigint generated by default as identity primary key,
  user_id text not null,
  model text not null,
  input_tokens integer not null default 0,
  output_tokens integer not null default 0,
  total_tokens integer not null default 0,
  created_at timestamptz not null default now()
);

create index if not exists usage_logs_user_id_created_at_idx
  on public.usage_logs (user_id, created_at desc);
```

如果你的项目只给这个私人助手使用，可以关闭 RLS：

```sql
alter table public.chat_history disable row level security;
alter table public.memories disable row level security;
alter table public.usage_logs disable row level security;
```

如果你启用了 RLS，需要为这三张表配置允许当前 Supabase Key 读写的策略。

## 表说明

### `chat_history`

保存每个用户的聊天记录。

- `user_id`：登录用户 ID
- `role`：`user` 或 `assistant`
- `content`：聊天内容
- `created_at`：创建时间

### `memories`

保存每个用户的长期记忆。

- `user_id`：登录用户 ID
- `memory`：记忆内容
- `created_at`：创建时间

### `usage_logs`

保存每次 OpenAI 请求后的 token 使用情况。

- `user_id`：登录用户 ID
- `model`：使用的模型
- `input_tokens`：输入 tokens
- `output_tokens`：输出 tokens
- `total_tokens`：总 tokens
- `created_at`：请求时间

## 联网搜索预留

页面里已经有“启用联网搜索”开关。目前代码中的 `get_search_context()` 是预留接口，后续可以在这里接入：

- Tavily
- SerpAPI
- Bing Search API

接入后，把搜索结果整理成简短上下文返回，应用会把它加入 system prompt。
